* 应用级缓存
    1. 缓存的作用 = 访问数据更快 
        * 热点数据
        * I/O瓶颈 = sql数据库 + 本地文件
        * 计算昂贵 =? 跨库
        * 5分钟法则 ?
        * 局部性原理 ?
        
    2. 缓存指标 = 命中率
    
    3. 回收策略 
        1. 空间
        2. 容量 = 数据条目
        3. 基于时间 = TTL TTI
        4. 基于java = 软引用【cg时如果内存不足，则回收】 + 弱引用【cg时如果发现，立即回收】
            * 必须在没有强引用指向的时候才可以
        5. 基于回收算法 
            * FIFO
            * LRU 最少使用的被删除 = 使用时间离现在最远的 常用 例如 Guava Cache、Ehcache
            * LFU 最小频率
